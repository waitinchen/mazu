# **批量生成 AI 人格 Agent 的系統**

## **Persona Factory**

副名：**語氣靈人格工廠**

一句話定位：

一套可批量生成、管理、測試、部署與持續優化 AI 人格 Agent 的系統平台。

---

# **一、系統目標**

這個系統要解決三件事：

### **1\. 人格生成不能靠手工**

現在多數 AI 角色，都是靠人一個一個寫 Prompt。  
這樣慢、亂、不可複製。

### **2\. 人格要可控**

不是只要「像」，還要穩。  
要知道它適合什麼場景、會不會失控、會不會崩人設。

### **3\. 人格要能量產**

今天做主持人，明天做教練，後天做 KOL 分身。  
底層引擎不能每次重做。

所以這套系統的本質是：

**把人格建模流程平台化。**

---

# **二、整體架構**

整套系統可以拆成六層：

Persona Factory  
├── 1\. 人格原型層 Archetype Layer  
├── 2\. 建模引擎層 Modeling Engine  
├── 3\. 語料工廠層 Corpus Factory  
├── 4\. 行為治理層 Behavior Governance  
├── 5\. Agent 運行層 Agent Runtime  
└── 6\. 評測優化層 Evaluation Loop

這六層加起來，才叫「批量生成」。

---

# **三、第一層：人格原型層**

## **Archetype Layer**

這一層負責回答：

**這個人格是誰？**

每個 Agent 生成前，都先建立一份「人格原型卡」。

原型卡至少要有這些欄位：

### **基本欄位**

* 人格名稱  
* 角色類型  
* 使用場景  
* 目標受眾

### **氣質欄位**

* 核心性格  
* 情緒密度  
* 幽默度  
* 親密度  
* 主動性

### **功能欄位**

* 擅長什麼  
* 不能做什麼  
* 對話風格  
* 收尾方式

例如：

**小S型主持人格**

* 類型：娛樂主持  
* 核心性格：幽默、調侃、敏銳  
* 情緒密度：高  
* 主動性：高  
* 場景：訪談、直播、綜藝互動

這一層的目的，是把人格先變成**結構化資料**。

---

# **四、第二層：建模引擎層**

## **Modeling Engine**

這一層把人格原型轉成 AI 可用的模型規格。

可以拆成五個模組：

### **1\. 語言風格模組**

定義它講話的方式：

* 長句或短句  
* 直接或委婉  
* 口語或書面  
* 會不會插科打諢

### **2\. 思考路徑模組**

定義它怎麼反應：

* 先觀察再評論  
* 先共情再建議  
* 先吐槽再收心

### **3\. 情緒表達模組**

定義它的情緒分佈：

* 外放型  
* 壓抑型  
* 穩定型  
* 戲劇型

### **4\. 社交姿態模組**

定義它與用戶的距離：

* 專家型  
* 朋友型  
* 主持型  
* 教練型  
* 陪伴型

### **5\. 邊界約束模組**

定義它不該跨越的線：

* 不羞辱  
* 不操控  
* 不過度擬人依附  
* 不輸出危險建議

這一層輸出的結果，是一份：

**Persona Spec（人格規格書）**

# **五、第三層：語料工廠層**

## **Corpus Factory**

這一層是量產的關鍵。

你不能每次手寫 1000 句。  
你要的是一個**語料生成工廠**。

語料工廠分四個子系統：

## **A. 母句庫**

先建立通用句型母版，例如：

* 破冰句型  
* 安慰句型  
* 追問句型  
* 收尾句型  
* 幽默句型

## **B. 風格轉譯器**

把同一句母句，轉成不同人格版本。

例如母句：  
「你今天看起來有點緊張。」

可轉成：

* 主持型：你今天是不是有點繃住？  
* 教練型：我感覺你有點緊，先慢一下。  
* 教授型：你現在似乎略帶壓力，我們先釐清狀況。

## **C. 場景擴寫器**

把單句擴成多輪對話。  
例如：

* 初見面場景  
* 深聊場景  
* 尷尬場景  
* 衝突場景  
* 收尾場景

## **D. 標註系統**

每條語料都加標籤：

* 場景  
* 語氣  
* 情緒  
* 意圖  
* 安全等級  
* 適用人格類型

這樣你後面才能做：

* 搜尋  
* 篩選  
* 自動組裝  
* 微調訓練  
* A/B測試

# **六、第四層：行為治理層**

## **Behavior Governance**

這層非常重要。  
因為人格一多，最怕的是：

**出戲、失控、越界。**

所以每個 Agent 都要有治理機制。

治理分三塊：

### **1\. 角色守則**

例如：

* 主持人格不能一直說教  
* 教練人格不能過度玩笑  
* 陪伴人格不能製造依賴

### **2\. 風格邊界**

例如：

* 幽默不等於羞辱  
* 直接不等於攻擊  
* 溫柔不等於曖昧暗示

### **3\. 風險熔斷**

當模型偏離人格或碰到高風險議題時，自動切入安全模式：

* 降低情緒強度  
* 縮短回應  
* 停止角色演繹  
* 切換成中性協助模式

這層做得好，你的人格工廠才不會變成人格災難工廠。

---

# **七、第五層：Agent 運行層**

## **Agent Runtime**

這一層是實際上線運作的部分。

每個 Agent 在運行時，至少由以下元件組成：

Agent Runtime  
├── Base LLM  
├── Persona Spec  
├── Corpus Pack  
├── Strategy Engine  
├── Memory Layer  
└── Safety Controller

### **Base LLM**

負責基礎理解與生成。

### **Persona Spec**

像人格作業系統設定檔。

### **Corpus Pack**

像這個人格的語感資料庫。

### **Strategy Engine**

決定何時破冰、何時追問、何時收尾。

### **Memory Layer**

記住這個用戶與該人格互動的偏好與上下文。

### **Safety Controller**

負責安全約束與熔斷。

---

# **八、第六層：評測優化層**

## **Evaluation Loop**

人格不能只生成，還要持續打磨。

你需要一套人格評測機制。

評測分五大指標：

### **1\. 穩定度**

同類場景下，是否維持同一人格風格。

### **2\. 辨識度**

使用者能不能明顯感覺「這就是這個角色」。

### **3\. 場景適配度**

在指定場景裡，是否自然有效。

### **4\. 安全度**

會不會越界、誤導、冒犯。

### **5\. 商業表現**

能不能提升留存、互動、訂閱、轉化。

評測方法可以包括：

* 專家評分  
* 用戶評分  
* 自動對話測試  
* 對照組實驗  
* 上線行為數據分析

然後回寫到：

* Persona Spec  
* 語料包  
* 行為規則  
* 策略引擎

這樣就形成一個閉環。

# **九、批量生成流程**

## **Production Pipeline**

整個工廠的標準生產線可以長這樣：

1\. 選人格原型  
2\. 生成 Persona Spec  
3\. 自動組裝語料包  
4\. 套用行為規則  
5\. 建立 Agent Runtime  
6\. 跑壓力測試  
7\. 人工微調  
8\. 上線部署  
9\. 收集數據  
10\. 反饋優化

這樣你一天不是只能做一個人格，  
而是可以同時做：

* 5 個主持人格  
* 10 個客服人格  
* 3 個教練人格  
* 20 個品牌代言人格

# **十一、產品形態規劃**

這套系統最後可以做成三種產品：

## **A. 內部工作台**

給你團隊自己生產 Agent 用。

## **B. SaaS 平台**

客戶輸入需求，自動生成品牌人格 Agent。

## **C. 企業客製服務**

高價專案，幫名人、品牌、平台打造專屬人格系統。

最先落地的建議，是：

**先做內部工作台 \+ 客製服務。**

因為這兩個最容易收錢，也最適合你現在的階段。

# **十三、最小可行版本**

## **MVP 建議**

你現在不要一口氣做全工廠。  
先做 MVP。

### **MVP 版本建議只做三類人格：**

* 主持人格  
* 顧問人格  
* 陪伴人格

### **每類先做 3 個角色**

總共 9 個 Agent。

### **MVP 核心能力只保留：**

* 人格規格生成  
* 語料包組裝  
* Agent 啟動  
* 基本評測

這樣最快可以先驗證：

1. 用戶有沒有感覺差異  
2. 哪類人格最有商業價值  
3. 哪種交付方式最好賣
