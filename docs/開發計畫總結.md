# Persona Factory 開發計畫總結

**專案代號：PF-01**  
**產品名稱：Persona Factory（語氣靈人格工廠）**  
**版本：v1.0**

---

## 一、專案定位與願景

### 1.1 一句話定位

**把名人的語氣、人格與主持能力，變成可長期運作的 AI 數位代理人；並建立一套可批量生成、管理、測試、部署與持續優化 AI 人格 Agent 的平台。**

### 1.2 核心價值

- **人格結構化**：人格設計不再依賴手工，可建模、可複製
- **語料自動化**：語料由工廠生成與風格轉譯，可檢索、可標註、可打包
- **Agent 批量化**：角色風格穩定、可評測、可治理，能量產部署

### 1.3 目標使用者

| 角色 | 職責 |
|------|------|
| Persona Designer | 設計人格原型 |
| AI Engineer | 模型與 Agent 部署 |
| Content Trainer | 語料與風格訓練 |
| Product Manager | 人格應用場景 |

## 二、系統架構總覽

### 2.1 數位名人引擎三層架構（DPHE）

```
數位名人引擎
├── 語氣靈人格層（Persona Layer）
├── AI 主持策略層（Host Strategy Layer）
└── 互動 Agent 層（Interaction Agent Layer）
```

### 2.2 Persona Factory 六層架構

```
Persona Factory Platform
├── 1. Persona Archetype Manager（人格原型管理）
├── 2. Persona Modeling Engine（人格建模引擎）
├── 3. Corpus Factory（語料工廠）
├── 4. Behavior Governance（行為治理）
├── 5. Agent Runtime（Agent 運行）
└── 6. Evaluation System（評測系統）
```

### 2.3 後端模組化單體建議（第一階段）

- **策略**：先做 **Modular Monolith**，不先拆微服務
- **目錄**：`app/` 下分 `api/`、`core/`、`db/`、`modules/`、`services/`、`workers/`、`integrations/`、`schemas/`、`utils/`
- **模組**：personas、corpus、agents、runtime、memory、evaluation、governance

## 三、技術棧與基礎建設

### 3.1 推薦技術

| 類別 | 技術選型 |
|------|----------|
| API | FastAPI |
| ORM | SQLAlchemy / SQLModel |
| 主庫 + 向量 | PostgreSQL + pgvector |
| 快取 / Queue | Redis |
| 背景任務 | Celery / Dramatiq / Temporal |
| Schema | Pydantic |
| Migration | Alembic |
| 容器 | Docker |
| 測試 | pytest |
| 觀測 | OpenTelemetry |

### 3.2 資料儲存

- **PostgreSQL**：主資料（personas、corpus、agents、sessions、messages、memory、evaluations、incidents）
- **pgvector**：語料與記憶的語意檢索
- **Redis**：session、cache、queue meta
- **Object Storage**：匯出檔、長文本報表、語音資產

## 四、開發階段與里程碑

### Phase 1：人格建模系統

- 完成：personas、persona_traits、persona_specs、prompt_templates
- API：建立/查詢/更新人格、更新 traits、生成 persona_spec
- 產出：人格原型 → Persona Spec（人格規格書）

### Phase 2：語料工廠

- 完成：corpus_items、corpus_packs、corpus_pack_items、corpus_embeddings、corpus_tags
- 功能：單筆/批量語料、風格轉譯、語料包組裝、語意搜尋、審核
- 產出：母句庫 → 風格轉譯 → 場景語料 → 語料包

### Phase 3：Agent 生成與 Runtime

- 完成：agents、agent_configs、agent_corpus_packs、sessions、messages
- 功能：建立 Agent、掛載語料包、配置、部署、建立 Session、對話回應
- 產出：可運行的 AI 主持人 / 人格 Agent

### Phase 4：記憶、評測與治理

- 完成：memory_items、memory_embeddings、evaluations、evaluation_metrics、incidents、deployments
- 功能：記憶寫入/查詢/語意搜尋、評測發起與報告、事件紀錄與狀態更新
- 產出：閉環優化與安全審計

## 五、MVP 範圍（v1.0）

### 5.1 必做功能

- Persona Archetype Manager
- Persona Modeling Engine
- Corpus Generator（含語料包）
- Agent Runtime（含 Session + 對話）
- 基本評測
- 基本 Governance（守則、熔斷、incident 紀錄）

### 5.2 MVP 必做 API（13 支）

| 領域 | API |
|------|-----|
| Persona | POST/GET personas、PUT personas/{id}/traits、POST personas/{id}/generate-spec |
| Corpus | POST corpus/items、POST corpus/generate、POST corpus/packs、POST corpus/packs/{id}/items |
| Agent | POST agents、POST agents/{id}/packs、POST agents/{id}/deploy |
| Runtime | POST runtime/sessions、POST runtime/respond |

### 5.3 延後項目

- 高級情緒識別
- 自動人格優化
- 跨人格互動

### 5.4 MVP 人格類別建議

- 主持人格、顧問人格、陪伴人格
- 每類先做約 3 個角色，共約 9 個 Agent，驗證差異感與商業價值

## 六、實作順序建議（對齊 Schema / API）

1. **Phase 1**：personas、persona_traits、persona_specs、prompt_templates
2. **Phase 2**：corpus_items、corpus_packs、corpus_pack_items、embeddings
3. **Phase 3**：agents、agent_configs、sessions、messages
4. **Phase 4**：memory、evaluations、incidents、deployments

---

## 七、語料庫設計要點（以小S型為例）

- **五大語氣庫**：破防、吐槽、情緒放大、自嘲、真誠
- **語料結構**：text、scenario、tone、emotion、intent、persona_type、safety_level
- **建議規模**：單一語氣靈約 1400+ 句可形成穩定風格
- **原則**：幽默 + 真誠並重，避免只有毒舌導致刻薄感

---

## 八、Runtime 流程（單次回應）

1. 讀取 session
2. 選擇策略（break_ice / tease / probe / amplify / close）
3. 檢索語料（corpus）
4. 檢索記憶（memory）
5. 組裝 prompt（persona_spec + corpus + context）
6. 呼叫 LLM
7. 安全檢查（governance）
8. 後處理
9. 寫入 message log

---

## 九、評測指標

- 人格穩定度
- 人格辨識度
- 場景適配度
- 安全性
- 用戶滿意度 / 商業表現（留存、互動、轉化）

## 十、微服務拆分時機（第二階段）

建議在以下情況再從 Modular Monolith 拆成微服務：

- Runtime QPS 明顯拉高
- Corpus 背景任務拖慢主服務
- Evaluation 跑批時間過長
- 治理策略變複雜、需獨立管控
- 團隊約 6–8 人以上，需更清楚模組邊界

**建議拆分服務**：gateway、persona-service、corpus-service、agent-service、runtime-service、memory-service、governance-service、evaluation-service

## 十一、商業模式與產品形態

### 11.1 產品形態

- AI Podcast 主持人
- AI 直播主持（TikTok、YouTube、小紅書等）
- AI 數位名人（粉絲與名人 AI 聊天）

### 11.2 商業模式

- 訂閱（粉絲與 AI 名人聊天）
- 節目授權（AI 主持節目）
- IP / 品牌授權

### 11.3 商業切入順序建議

1. **第一刀**：AI 主持 / 數位名人（辨識度高，貼合「語氣靈」）
2. **第二刀**：品牌人格 Agent
3. **第三刀**：陪伴型 Agent（市場大，治理要求高，後做）

## 十四、Repo 與部署建議

- **前期**：Monorepo（backend、worker、shared、docs、deployment）
- **環境**：local、staging、production（必要時加 sandbox）
- **CI/CD**：lint → type check → unit test → integration test → build image → deploy staging → smoke test → approval → deploy production

---

## 十五、一句話總結

**Persona Factory 開發計畫** 的目標是：  
先以 **模組化單體** 把人格原型、語料工廠、Agent 組裝、Runtime、治理與評測六大能力做穩；再依流量與團隊規模，沿責任邊界拆成微服務，形成 **可設計、可量產、可部署、可評測、可治理的 AI 人格生產基礎設施**。
